    1. Introduccion

    El presente documento constituye la memoria tecnica del proyecto "Interfaz Web" desarrollado para LuxFrame Studio, un portfolio fotografico especializado en naturaleza, paisaje y astrofotografia. El proyecto tiene como objetivo principal diseñar, desarrollar y validar una interfaz web profesional, responsive, accesible y optimizada para la visualizacion de contenido multimedia de alta calidad.

    Objetivos del proyecto

    • Crear una web corporativa para el fotografo Ismael Tayllefert que muestre su portfolio de manera atractiva y funcional.
    • Aplicar buenas practicas de desarrollo web: HTML semantico, CSS modular con Sass, JavaScript limpio y accesibilidad WCAG.
    • Optimizar el contenido multimedia (imagenes WebP, video MP4/WebM, audio MP3/OGG) para garantizar tiempos de carga rapidos sin sacrificar calidad.
    • Implementar elementos interactivos avanzados: carrusel personalizado, pestañas, acordeones, modales, tooltips y animaciones con Canvas API.
    • Validar la accesibilidad y usabilidad del sitio mediante herramientas externas y rubricas especificas.

    Alcance

    El proyecto abarca desde la planificacion inicial hasta la entrega final, incluyendo diseño visual, desarrollo frontend, optimizacion multimedia, testing de accesibilidad y documentacion tecnica.

    HTML5 Bootstrap 5.3 Sass (SCSS) CSS3 JavaScript ES6+ Canvas API Bootstrap Icons Google Fonts

    3.2 Estructura HTML y Bootstrap 5.3

    Se ha aplicado una estructura HTML5 semantica con etiquetas apropiadas para cada seccion:

    <header> → <nav> → <main> → <section> → <article> → <footer>

    Uso estrategico de Bootstrap:

    • Rejilla responsive: container, row, col-* para adaptar el diseño a movil, tablet y escritorio.
    • Componentes nativos: Navbar fija con scroll, carrusel fade, pestañas para servicios, acordeon para proceso, modal para detalles de portfolio, tooltips para equipo.
    • Clases utilitarias: d-flex, gap-3, text-center, mb-4 para reducir CSS personalizado.

    3.3 Estilos personalizados con Sass

    El archivo custom.scss organiza los estilos en secciones logicas:

    // Variables de diseño (design tokens)
$color-accent: #c8a96e;
$font-display: 'Playfair Display', Georgia, serif;

// Mixins reutilizables
@mixin card-base {
  background: $color-surface;
  border: 1px solid $color-border;
  border-radius: $radius-md;
  transition: transform $transition, box-shadow $transition;
}

// Componentes
.portfolio-card {
  @include card-base;
  &:hover { transform: translateY(-6px); }
}

    4. Parte 3 y 4: Contenido Multimedia e Interactividad

    4.1 Gestion de imagenes

    Se han integrado 20 fotografias de naturaleza del fotografo Ismael Tayllefert, optimizadas para web:

    Proceso de optimizacion:

    1. Seleccion: Imagenes originales en RAW/JPG de alta resolucion.
    2. Edicion: Ajustes basicos en Lightroom (exposicion, contraste, recorte).
    3. Conversion: Exportacion a WebP con Squoosh.app (calidad 85%, reduccion ~40% de peso).
    4. Fallback: Generacion de version JPG para navegadores antiguos.
    5. Implementacion: Uso de <picture> con srcset para carga condicional.

    Ejemplo de implementacion:

    <picture>
  <source srcset="media/Hojas_Rojizas_en_la_Sombra.webp" type="image/webp">
  <img src="media/Hojas_Rojizas_en_la_Sombra.jpg" 
       alt="Hojas con tonos rojizos, morados y amarillentos sobre fondo oscuro" 
       loading="lazy" width="800" height="600">
</picture>

    4.2 Video y audio

    Video showreel (12 segundos):

    • Contenido: Secuencia de agua fluyendo en entorno natural.
    • Formatos: MP4 (H.264) como principal, WebM (VP9) como alternativa.
    • Optimizacion: Codificado con HandBrake (CRF 23, audio AAC 128kbps, resolucion 1280x720).
    • Accesibilidad: Subtitulos en formato WebVTT (showreel-es.vtt) con descripcion de audio.

    Audio ambiental:

    • Contenido: Sonidos de naturaleza (agua, viento, pajaros).
    • Formatos: MP3 (128kbps) como principal, OGG como fallback.
    • Implementacion: Reproductor personalizado con controles accesibles y barra de progreso animada.

    4.4 Animaciones con Canvas API

    Se han desarrollado tres animaciones generativas utilizando la API Canvas de HTML5:

    1. Hero Canvas (Constelacion de estrellas): Fondo animado con estrellas que se conectan mediante lineas cuando estan cercanas. Particulas con movimiento vertical, calculo de distancias, gradientes lineales.
    2. Particle Canvas (Seccion Multimedia): Particulas doradas que flotan y rebotan en los bordes, simulando luz natural. Velocidad aleatoria, gradientes radiales para efecto bokeh.
    3. Process Canvas (Diagrama de flujo): Animacion secuencial que ilumina los pasos del proceso al entrar en viewport. IntersectionObserver para trigger, dibujo progresivo con lineDash.

    4.5 Justificacion de formatos multimedia

    Imagenes:

    Formato Uso Justificacion
    WebP Principal Compresion superior a JPG (~30% menos peso) con misma calidad visual. Soportado por el 98% de navegadores modernos.
    JPG Fallback Compatibilidad universal con navegadores antiguos (< Chrome 85, Safari <14).
    SVG Logo, iconos Escalado infinito sin perdida, peso minimo, editable con CSS.

    Video:

    Formato Uso Justificacion
    MP4 (H.264) Principal Soporte universal en navegadores y dispositivos. Buen equilibrio calidad/peso.
    WebM (VP9) Alternativa Mejor compresion que H.264 en navegadores compatibles (Chrome, Firefox, Edge).

    Audio:

    Formato Uso Justificacion
    MP3 Principal Soporte universal, buena calidad a 128-192 kbps.
    OGG (Vorbis) Fallback Formato abierto, mejor calidad a mismo bitrate en Firefox/Chrome.

    Buenas practicas aplicadas:

    • Lazy loading: Atributo loading="lazy" en imagenes fuera del viewport inicial.
    • Dimensiones explicitas: width y height en <img> para evitar CLS.
    • Fallbacks multiples: <picture>, <video> con multiples <source>.
    • Accesibilidad multimedia: Subtitulos VTT para video, transcripcion de audio en esta memoria.

    5. Parte 5: Accesibilidad y Usabilidad

    5.1 Principios de accesibilidad aplicados

    El proyecto sigue las Pautas de Accesibilidad para el Contenido Web (WCAG) 2.1, nivel AA. Se han aplicado las siguientes tecnicas:

    Texto alternativo y estructura semantica

    • Todas las imagenes informativas incluyen atributo alt descriptivo.
    • Imagenes decorativas usan aria-hidden="true" o alt="".
    • Estructura de encabezados jerarquica: <h1> unico en hero, <h2> para secciones, <h3> para subtitulos.
    • Uso de etiquetas semanticas: <nav>, <main>, <section>, <article>, <footer>.

    Contraste de color

    • Texto principal (#f0eeea) sobre fondo oscuro (#0d0d14): ratio 12.5:1 (WCAG AAA).
    • Texto secundario (#b0b0c0) sobre fondo oscuro: ratio 7.1:1 (WCAG AA/AAA).
    • Enlaces y acentos (#c8a96e) sobre fondo: ratio 6.8:1 (WCAG AA).

    Navegacion por teclado

    • Todos los elementos interactivos son alcanzables con Tab.
    • Indicador de foco visible: :focus-visible { outline: 2px solid var(--color-accent); }.
    • Skip link para saltar al contenido principal (accesible al enfocar con teclado).

    Formularios accesibles

    • Cada <input> tiene <label> asociado mediante for/id.
    • Campos requeridos marcados con aria-required="true" y asterisco visible.
    • Mensajes de error claros y asociados al campo mediante aria-describedby.

    Contenido multimedia accesible

    • Video con subtitulos WebVTT (showreel-es.vtt) que incluyen descripcion de audio.
    • Audio con transcripcion textual en esta memoria.
    • Controles de video/audio con etiquetas ARIA descriptivas.

    6. Incidencias y Conclusiones

    Incidencias encontradas y soluciones

    Incidencia Descripcion Solucion aplicada
    Contraste de texto en fondos oscuros La clase .text-muted de Bootstrap tenia contraste insuficiente sobre fondos oscuros. Sobrescritura con !important: .text-muted { color: #b0b0c0 !important; } → ratio 7.1:1.
    Imagenes de portfolio no llenaban el contenedor Las tarjetas mostraban imagenes con espacios vacios. CSS: .portfolio-card-img img { width:100%; height:100%; object-fit:cover; }.
    Subtitulos de video no cargaban Error CORS al abrir HTML con file://. Solucion: usar servidor local para testing (python -m http.server o Live Server en VS Code).
    Animaciones canvas en movil Rendimiento bajo en dispositivos antiguos. Optimizacion: reducir numero de particulas en movil, usar passive: true en event listeners.
    Formulario sin validacion visual Los errores no se mostraban claramente. Integracion de Bootstrap Validation + mensajes personalizados con invalid-feedback.

    Conclusiones personales

    1. Importancia de la planificacion: Una estructura clara de tareas y versionado ha permitido organizar el trabajo y mantener un historial de cambios, facilitando la depuracion y el trabajo iterativo.
    2. Bootstrap como base solida: La framework ha acelerado el desarrollo de componentes responsive, pero ha requerido personalizacion cuidadosa para mantener la identidad visual de LuxFrame.
    3. Accesibilidad no es opcional: Implementar WCAG AA ha mejorado la experiencia para todos los usuarios, no solo para aquellos con discapacidades. Herramientas de validacion son esenciales.
    4. Optimizacion multimedia es critica: La diferencia de peso entre imagenes RAW y WebP optimizadas es abismal (~70% menos). Esto impacta directamente en tiempos de carga y SEO.
    5. Canvas API potencia la experiencia: Las animaciones generativas añaden valor visual sin depender de librerias pesadas, pero requieren testing en distintos dispositivos.
    6. Documentacion es parte del producto: Una memoria bien estructurada no solo cumple requisitos academicos, sino que facilita el mantenimiento futuro y la colaboracion.
